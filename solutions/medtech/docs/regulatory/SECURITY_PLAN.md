## 1. Purpose

This document defines the cybersecurity plan for the medical device software, addressing FDA premarket cybersecurity guidance and IEC 62443 requirements.

## 2. Threat Model

### 2.1 Asset Inventory

| Asset | Sensitivity | Availability Requirement |
|-------|------------|------------------------|
| Patient data | High (PHI) | High |
| Device configuration | Medium | Medium |
| Audit logs | High (compliance) | High |

### 2.2 Threat Categories

| Category | Threats | Mitigations |
|----------|---------|-------------|
| Data breach | Unauthorized access to PHI | Encryption, RBAC, PII detection |
| Tampering | Modification of device config | Immutable audit log, approval gates |
| Denial of service | System unavailability | Rate limiting, health monitoring |
| Repudiation | Untracked changes | Trace IDs, 21 CFR Part 11 signatures |

## 3. Security Controls

### 3.1 SAGE Built-in Controls

| Control | Implementation |
|---------|---------------|
| PII Detection | Presidio-based scrubbing (redact/mask/flag) |
| RBAC | Role-based access via `/access-control` |
| Audit Trail | Immutable SQLite log with trace IDs |
| HITL Approval | All agent proposals require human approval |
| Data Isolation | Per-solution `.sage/` directories |
| Electronic Signatures | 21 CFR Part 11 compliant signatures |

### 3.2 Infrastructure Controls

| Control | Implementation |
|---------|---------------|
| Transport encryption | HTTPS/TLS |
| Authentication | API key + session management |
| Network isolation | Configurable air-gap deployment |
