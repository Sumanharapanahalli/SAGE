## 1. Purpose

This document defines the change control procedure for the medical device software, per IEC 62304:2006+A1:2015 Clause 6.1 and FDA 21 CFR Part 820.30(i).

## 2. Change Request Process

```
Change Request → Impact Assessment → Approval → Implementation → Verification → Release
```

### 2.1 Change Categories

| Category | Risk | Approval Required |
|----------|------|-------------------|
| Critical | Safety-related | Regulatory + Engineering |
| Major | Functional change | Engineering Lead |
| Minor | Non-functional | Developer |
| Cosmetic | UI/docs only | Auto-approve |

## 3. Impact Assessment

Each change request must assess:
- Regression risk (which existing functions may break)
- Testing impact (which test cases need re-execution)
- Documentation impact (which regulated docs need update)
- Traceability impact (which requirement links affected)
