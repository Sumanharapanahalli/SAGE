# SAGE — Deep Full-Repo Gap Review (2026-07-14, evening)

A 14-agent fleet (product tier + framework tier) swept the whole repo with live probes. The
API session limit killed the verifier phase, so **findings were recovered from the finder
journals (60 distinct) and the load-bearing ones re-verified by hand** in the main loop —
which, given this session's proven lesson that critics fabricate ~half their "critical"
findings, is the stronger check anyway. This run's finders were unusually high-quality: most
came with pasted live-probe output and exact `file:line`.

**Headline: the flashy "AI writes code → you approve → the code lands" pipeline is broken
end to end.** Every engine's happy-path *UI* works and every LLM call is real, but no engine
currently delivers its end artifact. Six blockers describe facets of this one fault. **Three
are fixed and pushed today; three are architectural and sequenced below.**

| B4 | **No external-repo code awareness.** Nothing points agents at an imported project's code: `project.yaml` has no repo/path field; `onboarding.generate_solution` takes no code dir; `CodingAgent._ROOT` and the developer repo-map both resolve to SAGE's own checkout. An imported pose-engine gets *log analysis only* — the analyst can say "guard `one_euro_filter.cpp:88`" but nothing can open or patch that file. **This is the operator's actual goal and it does not exist.** | OPEN — architectural (Evolve Level 0). |
| B5 | **BuildOrchestrator writes no code to disk.** A "completed" build runs a real planner + 3-critic + N-task LLM pipeline and produces a README, an AGENTS.md, a 4000-char diff preview, and a list of filenames that were never created. The only copy of generated code is an unread JSON blob in `data/build_checkpoints.db`. | OPEN — terminal stage (`_integrate`/`_finalize`) is a stub. |
| B6 | **Approved code_diff commits an empty worktree.** CodingAgent writes to the MAIN tree; the isolation worktree is created *afterward* from HEAD (so it lacks the changes); approval commits in that empty worktree → `committed:false` while the proposal shows "approved." No merge-back path exists anywhere. | OPEN — worktree lifecycle inverted. |

B4–B6 are one job: **give agents a real, isolated workspace that is the target project, write
files into it, and apply approved changes back to it.** Patching them piecemeal will not work;
they need the Evolve importer's workspace model. B4 is the linchpin — it is both the missing
feature and the foundation the others stand on.

## Major (verified by hand where marked ✓; others are high-confidence finder claims)

**Compliance integrity — the regulated-industry moat, currently hollow:**
- ✓ **Every audit row is unsigned (0/529).** `verification_signature` is a column the only
  writer never populates (`audit_logger.py:240-251`); a HMAC hash-chain module exists but is
  unwired. Docs claim FDA 21 CFR Part 11 e-signature compliance. SAGE would fail its own gap
  analysis. (`M3` from the morning review — now confirmed at runtime.)
- ✓ **RBAC is decorative.** Desktop `approvals.approve` never consults `proposal.required_role`;
  an `admin`-gated proposal is approved by anyone. Web checks a client-typed name and fails open.
- **The audit signer has no UI** — `operator.set` is registered at every layer but no page
  calls it, so every approval is signed with the fallback `"operator"`.
- **HIL mock mode mints all-PASS "IEC 62304 Test Evidence"** into the audit log (desktop
  default) — fabricated compliance records.

**Config / provider safety:**
- ✓ **"Save as default" on LLM switch corrupts `config.yaml`** — its `re.sub(r'^(\s*provider\s*:...)')`
  matches BOTH `llm.provider` and `auth.oidc.provider`, silently rewriting the SSO provider.
- **`llm.switch` accepts unknown providers** — a typo (`"olama"`) reports success and lands you
  on the Claude API with no key; every generation then fails.
- **Ollama default `gemma3:1b` isn't installed**; a 404 is mis-reported as "server unreachable."
- **Costs page bills free local Ollama at paid rates** — the `$0 "ollama"` price row can never
  match (pricing keys on model name, receives `llama3.2`/`gemma3:1b`).

**Domain runners — the gym gradebook is largely measuring prose, not execution:**
- **openbrowser fabricates full QA reports** via an explicit "generate realistic browser test
  results" LLM prompt when its binary is absent (always, on Windows).
- **opensim never invokes a simulator; openeda's DRC verification is dead code** (reads keys
  never written) even with KiCad installed.
- **The "3-tier isolation cascade" is isolation theater** — Tier 2 is `shell=True` on the host;
  openterminal pipes unfiltered LLM commands into a host tmux with no HITL gate. *No security-
  conscious operator should enable it.*
- Net: the developer's 80–91 gym scores were LLM-judge + keyword + syntax-compile — **no test
  suite ever actually passed during grading on this machine.**

**Dead code (built, tested, unreachable at every UI layer):** `agentrun` (agent_hire — a
SOUL.md proposal kind), `safety` (FMEA/FTA/ASIL), `health` preflight (*still* unregistered
after yesterday's fix to it), `mr` (GitLab), `queuedb` (task history), `compliance.flags`.

**LangGraph:** installed, workflows shipped, but a `sys.modules` loader bug kills the SWE/HIL
workflows and a missing checkpointer makes every HITL interrupt gate silently report
"completed" while skipping the post-approval node.

## Open operational gaps (O-series)
- **O1. No desktop TaskWorker** — even with B3's fix, queued tasks are visible but nothing
  drains the queue. The "agents do the work" promise needs a worker (or route through `jobs.py`).
- **O2. No vector-store backup/rebuild** — a corrupt `chroma.sqlite3` silently drops to an
  empty keyword fallback; compounding memory is unrecoverable.
- **O3. Desktop app has no CI** (its 1011 tests run in no job; CI tests only `web/`).
- **O4. Git-tracked demo runtime data** fights the archive feature and churns cross-machine.

## Refuted during review (recorded so they don't return)
- Analyst DOES do memory retrieval (`analyst.py:48`). Critic prompts ARE bounded (`[:10000]`/`[:8000]`).
  The 9 orchestrator-intelligence modules are NOT shelfware (all have non-test importers;
  `routes/orchestrator.py` mounted at `api.py:150`). `make verify` did not break. Cost tracking
  hooks are provider-agnostic (the *pricing table* is the bug, not the capture).

## The honest sequence to "fully usable"
1. ✅ **B1** destructive revert (done — it was actively dangerous).
2. ✅ **B2/B3** store unification (done — AI work is now reachable/visible on desktop).
3. **O3** desktop tests into CI — an afternoon; stops silent regressions permanently.
4. **B4** the external-project importer (Evolve Level 0) — the operator's goal and the
   foundation for B5/B6. Biggest single build; do it as its own scoped project.
5. **B5/B6 + O1** on top of B4's workspace model: write build output to disk, invert the
   worktree lifecycle, add a TaskWorker — so approved work actually lands.
6. **Compliance pass**: real operator identity + signature + enforced RBAC (the moat).
7. Config-safety papercuts (LLM-switch validation, the SSO-corruption regex, Ollama default),
   then register-or-delete the dead handlers.

Nothing here is fatal to the vision; it is the gap between an impressive demo surface and a
tool that keeps its promises. The three fixes shipped today closed the dangerous and the
loop-breaking ones. The rest is real, scoped, and sequenced.
