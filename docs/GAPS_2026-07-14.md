# SAGE — What Still Blocks Full Usability (2026-07-14, full-repo review)

Scope: the **whole repo** — framework (`src/`), web stack (`api.py` + `web/`), desktop app,
gym/loop, solutions, deploy artifacts. Everything below was probed in this session; claims
that could not be evidenced were dropped, and several candidate findings were **refuted**
during verification (listed at the end, because false gaps cost as much as real ones).

Method note: a 15-agent verification fleet was launched and killed by the API session limit,
so the probes were run directly. Each finding cites what was actually observed.

## Blockers — core flows that cannot be completed today

### B1. There is no way to import an existing project
The operator's stated goal is to run SAGE on **pose-engine**. Today that is impossible:
`onboarding.generate_solution(description, solution_name, compliance_standards, integrations,
parent_solution, org_name, org_context)` — there is **no parameter for an existing code
directory**. It scaffolds fresh YAML only. `solutions.list` scans one directory for
`project.yaml`/`SKILL.md` markers. The manual path (hand-write 3 YAMLs inside the target
repo, set `SAGE_SOLUTIONS_DIR`) is undocumented and unvalidated.
*This is Level 0 of the SAGE Evolve spec (docs/superpowers/specs/2026-07-13), which is
designed and debated but **unbuilt**.*

### B2. Approved work is executed — but the task queue has no worker
Better than feared, worse than needed. Verified: `approvals.approve` **does** execute
approved proposals (`handlers/approvals.py:274 → proposal_executor.execute_approved_proposal`
via a jobs system) with executors for **15 action types** (`proposal_executor.py:456-472`:
yaml_edit, knowledge_*, code_plan, code_execute, code_diff, implementation_plan, agent_hire,
mcp_invoke, onboarding_generate, …). But the **tasks.yaml queue** — the thing wave scheduling
and "agents do the work" rests on — has **no TaskWorker in the desktop**: `queue.retry`
flips a task to pending in `.sage/queue.db` and *nothing drains it* (verified when
cancel/retry was built). And a build mid-`building` has **no abort path**. So: propose →
approve → execute works for single proposals; sustained agent *work* does not.

### B3. The maintained UI has zero CI coverage — the abandoned one is fully tested
`.github/workflows/ci.yml` `frontend-tests` runs `npm install`, `tsc`, `vitest`, and
Playwright **in `web/`**. The desktop app — where all current development happens, with
**601 sidecar + 410 vitest tests** — appears in **no CI job at all**. Every desktop
regression ships silently; this is the mechanical root of "features rot without anyone
noticing." (`make verify`'s 8 gates are also not in CI.)

## Major — works only with workarounds, or breaks the product's promises

### M1. Remove can resurrect a ghost, and git-tracked demo data fights the feature
Two interacting defects observed live today:
- `four_in_a_line` is **tracked in git** (via the `.gitignore:88` negation), so archiving it
  left `git status` full of `D solutions/four_in_a_line/*.yaml` — a user could commit the
  deletion by accident, and the tracked `chroma.sqlite3` churns/conflicts across laptops.
- Any stale reference to a removed solution **silently recreates it**: the sidecar's
  `_wire_handlers` does `sage_dir.mkdir(parents=True)`, so a verify probe (observed) or the
  auto-reopen of `lastSolution` on another machine resurrects an **empty husk** of the
  removed solution — which reads as data loss.

### M2. Five handlers are still dead code
Confirmed unimported in `app.py`: **agentrun** (run/hire agents — `agent_hire` is a SOUL.md
proposal kind that even *has an executor*, yet no UI can propose it), **mr** (GitLab review
loop), **queuedb** (task history/subtasks), **safety** (FMEA/FTA), and — ironically —
**health** (the preflight check *fixed yesterday*, tested, still never registered).

### M3. The compliance signature is a placeholder
`audit_logger.py:71`: `verification_signature TEXT, -- Placeholder for digital signature` —
**never written**. `approver` is client-supplied free text; `operator.yaml` defaults to
empty; `required_role`/`approver_role` are **enforced nowhere** in the sidecar (zero grep
hits in approvals.py). The audit trail is honest *logging*, but for the regulated-industry
promise (21 CFR Part 11 expects e-signatures) it is not yet a compliance record. SAGE's own
regulatory checklist demands "secure audit trail (immutable, timestamped)" — SAGE itself
would fail its own gap analysis here.

### M4. Three-and-a-half frontends/backends, no statement of which is canonical
- `web/` — CI-tested, feature-drifted, barely maintained.
- `sage-desktop/` — maintained, not CI-tested (B3).
- `web/src-tauri/` — a third shell wrapping the web UI.
- **`backend/`** — an entire parallel FastAPI backend (own Dockerfile, alembic, models)
  imported by **nothing** in `src/` or `web/` (only greppable match is a code comment).
  An abandoned second implementation shipping at repo root misleads every newcomer — and
  every LLM agent that scans the repo.

### M5. Out-of-the-box defaults are broken or false
- `Makefile: PROJECT ?= iot_medical` — **no such solution exists**; `make run` fails on a
  fresh checkout.
- CLAUDE.md says "Default: Gemini CLI (no API key)"; `config.yaml` ships
  `provider: claude-code`. One of them is lying to the new user.
- `.mcp.json` hardcodes `/home/shetty/sandbox/SAGE` — a path from someone else's machine.
- `solutions/` contains junk: empty `sol_a`/`sol_b` dirs appear in the picker's scan path.

### M6. Synchronous LLM calls freeze the whole desktop app
The sidecar's NDJSON loop is single-threaded; `analyze.run` and similar RPCs call CLI LLMs
inline at 10–180s latency (Gemini's measured curve: 5KB→7s, 50KB→142s). While one runs,
**every page of the app is blocked**, with no progress events and no streaming. The jobs
system (used by builds/approvals) exists but slow RPCs aren't routed through it.

### M7. The gym cannot measure SAGE's own agents, and isn't reachable from the UI
No runner exists for `planner`, `monitor`, `universal`, `critic`, `product_owner`, `coder` —
the framework's own roster is unmeasurable, and the analyst (the weakest measured role at
47.9) can't be trained where it matters. The gym runs from scripts only; the desktop Eval
page lists suites that nothing ships (`eval.list_suites → []` on every solution).

### M8. The self-improvement catalogue is partly unrunnable as shipped
`scripts/self_improve.py` tasks that feed whole large files as context (task 1: the 232KB
`api.py`) now trip the 40KB oversize evaluator guard — correctly, since that exact task
burned 5 hours producing a phantom 0.0. The catalogue needs re-scoping into judgeable units.

## Minor — friction

- **Feature-matrix drift**: chat/execute, undo, subtasks/waves, connectors, distillation
  exist only in the web API; activity, regulatory, queue control, console, knowledge-sync
  only in the desktop. A user told "SAGE has X" has a coin-flip chance of the wrong UI.
- **Monitor is inert by default** — `monitor.status: running: false`, nothing feeds it
  signals from the desktop.
- **Workflows page is empty for every shipped solution** except `medtech_team` (the only one
  with a `workflows/` dir); langgraph *is* installed, the feature just looks dead.
- **Deploy artifacts unvalidated**: docker-compose/helm/k8s reference the pre-split layout
  in places; none are built in CI. (Flagged, not fully verified this pass.)
- **`__collective__/` and `org.yaml` live inside `solutions/`** — surprising locations that
  the picker and tooling must special-case.

## Deliberately deferred (documented decisions, not new gaps)
Installer packaging (sidecar/Python not bundled — `make desktop-dev` is the supported path),
auth/multi-user, response streaming, Integrations/Connectors pages, Distillation, HIL against
real hardware.

## Raised during review and REFUTED — recorded so they don't come back
- "The analyst skips CONTEXTUALIZE (memory search)" — **false**: `analyst.py:48`
  `vector_memory.search(log_entry, k=2)`; feedback write-back at :104.
- "Critic prompts are unbounded like the 5-hour loop hang" — **false**: artifacts capped
  (`critic.py:167` `[:10000]`, diffs `[:8000]`).
- "The 9 orchestrator-intelligence modules are shelfware" — **overstated**: all nine have
  non-test importers (depth of integration varies; none is a dead file).
- "`make verify` broke when the demo solution was archived" — **false**: 8/8 (it self-heals
  by recreating the dir — which is finding M1's mechanism, not a health signal).
- "Cost tracking skips CLI providers" — **unsupported**: `cost_tracker` hooks sit in the
  provider-agnostic gateway paths (`llm_gateway.py:1287,1415`).

## The shortest path from here to "fully usable"

1. **B1** — build the solution-importer (Evolve Level 0): point SAGE at an existing repo,
   propose the 3 YAMLs + component map through the HITL gate. Unblocks pose-engine.
2. **B3** — add desktop suites + `make verify --json` to CI today (an afternoon, huge return).
3. **B2** — a TaskWorker for the desktop queue + build abort.
4. **M1/M2** — untrack demo runtime data; guard the mkdir-resurrection; register or delete
   the five dead handlers (delete is a valid answer).
5. **M3** — real operator identity + signature on approvals (the compliance moat).
